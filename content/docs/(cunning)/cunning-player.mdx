<Lang lang="en">

**Cunning Player** is the **Web Runtime** for the Cunning3D ecosystem. Thanks to Rust's perfect support for **WebAssembly (WASM)**, we have compiled the entire computation core to run inside the browser.

This means users can load, tweak, and preview complex CDA assets in real-time via a simple web link, **without installing any software**.

</Lang>

### ğŸŒ Client-Side Compute
Unlike cloud rendering solutions, all Cunning Player geometry computations happen **locally in the user's browser**.
- **Zero Server Cost**: No expensive GPU instances needed to bake models.
- **Privacy**: Asset logic executes locally.
- **Offline Capable**: Once loaded, parameters can be tweaked even without an internet connection.

### ğŸ§µ Web Worker Multithreading
To prevent heavy geometry operations from freezing the browser page, we utilize **Web Workers** to offload Cook tasks to background threads. Even with million-polygon mesh generation, the page UI remains responsive.

### ğŸ¨ WebGPU / WebGL2 Rendering
Reusing the Bevy Engine render pipeline, Cunning Player delivers high-fidelity visuals in the browser consistent with the desktop app.

- **Real-time Wireframe Overlays**
