## Header

| Offset | Size | Field | Value |
|--------|------|-------|-------|
| 0 | 4 | Magic | `CDA\0` |
| 4 | 4 | Version | `0x00000001` |
| 8 | 4 | Chunk Count | `2` |

## Chunk Table

Each entry (12 bytes):

| Offset | Size | Field |
|--------|------|-------|
| 0 | 4 | Tag (e.g., `EDIT`, `GAME`) |
| 4 | 4 | Offset from file start |
| 8 | 4 | Size in bytes |

`hud_units` and `coverlay_units` declare which interaction units are exposed by the CDA. They describe **exposure + defaults**, without tying to any specific UI stack.

## [GAME] Chunk

Compact runtime representation. JSON format:

```json
{
  "uuid": "48a309f5-77e9-4feb-ade2-d99344bb28f7",
  "ops": [
    { "id": 0, "code": 1, "params": {...} },
    { "id": 1, "code": 11, "params": {...} }
  ],
  "topology": [[0, 1], [1, 2]],
  "params": [...],
  "inputs": [...],
  "outputs": [...]
}
```

| Field | Description |
|-------|-------------|
| `ops` | Operator nodes with OpCode |
| `topology` | Execution order |
| `params` | Exposed parameter definitions |
| `inputs` | Input port specifications |
| `outputs` | Output port specifications |

## OpCodes

| Code | Operator |
|------|----------|
| 1 | Input |
| 2 | Output |
| 10 | Boolean |
| 11 | PolyExtrude |
| 12 | PolyBevel |
| 13 | GroupCreate |
| 20 | Merge |
| ... | ... |

## Design Goals

| Goal | Implementation |
|------|----------------|
| Human-readable | JSON chunks |
| Git-friendly | Text-based, no binary blobs |
| Compact runtime | Separate `[GAME]` chunk |
| Editor roundtrip | `[EDIT]` preserves full graph |

<Callout type="info">
Use hex editor or `cunning_cda_dump` tool to inspect CDA files.
</Callout>
